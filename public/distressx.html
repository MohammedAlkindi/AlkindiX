DistressX

A research project on financial distress prediction using interpretable machine learning.
Built to prioritize clarity, traceability, and decision usable outputs.

View artifacts Back to AlkindiX

Overview

DistressX studies corporate financial distress prediction using structured financial ratios and interpretable models. The goal is to build a pipeline that supports risk stratification while remaining auditable and explainable.

Research question

Can we predict financial distress reliably while preserving interpretability for governance and decision support.

Core principle

Prefer transparent reasoning over marginal metric gains when deployment requires accountability.

Data

The dataset consists of firm year observations with financial ratios across profitability, leverage, liquidity, and efficiency. Features are standardized and validated to reduce leakage and improve comparability across firms.

Scale Approximately 105,000 firm year observations

Features 60 plus financial ratios and derived indicators

Methods

Pipeline

  • Preprocessing with ratio standardization and quality checks
  • Unsupervised clustering to identify risk regimes
  • Cluster aware logistic regression for interpretable predictions
  • Benchmarking against higher capacity models for comparison

Interpretability strategy

The primary model is selected to preserve coefficient level interpretability and enable transparent reporting of key drivers.

Evaluation

Metrics include accuracy, ROC AUC, calibration checks, and stability across regimes.

Results

Interpretable model About 0.84 ROC AUC with clear coefficient drivers

Benchmark model About 0.90 ROC AUC using XGBoost for comparison

The project explicitly documents the performance versus transparency tradeoff. The selected model favors interpretability for governance aligned use cases.

Primary drivers

  • Retained earnings related indicators
  • EBIT to total assets style profitability measures
  • Leverage ratios
  • Expense ratios

Limitations

  • Label definitions may vary by jurisdiction and reporting standards
  • Distress events can be rare and can bias metrics without careful validation
  • Model transfer across industries and time periods requires monitoring
  • Deployment should include calibration checks and governance review
